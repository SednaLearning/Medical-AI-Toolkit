# Prompts

Dieser Ordner enthält wiederverwendbare KI-Prompts für regulatorische und technische Aufgaben.

## Struktur eines Prompts

Jeder Prompt basiert auf:

- Kontext
- Ziel
- Rolle
- Aufgabe
- Regeln
- Gewünschtes Ergebnis
- Beispiele
- Hinweise

## Benennung

Dateinamen sollen den Zweck des Prompts beschreiben.

Beispiele:

- ISO14971_Risikomanagementbericht_Pruefung.md
- ISO14155_Pruefung_Prueferbroschuere.md
- MDR_klinische_Bewertung_Pruefung.md

## Entwicklung eines Prompts

Ein Prompt durchläuft folgende Schritte:

1. Entwurf
2. Test
3. Verbesserung
4. Freigabe

## Testdokumentation

Für wichtige Prompts sollen dokumentiert werden:

- verwendetes Testdokument
- erkannte Schwächen
- Verbesserungen
- bekannte Einschränkungen
